# QR Code Simple - 自定义一维码解码器契约

本文档记录 Pharmacode、Plessey、MSI Plessey、Telepen 四种自定义一维码的解码规则，用于指导生成器实现。

---

## 1. Pharmacode

### 编码规则
- 表示一个整数，范围 **3..131070**。
- 从最低位开始，反复判断奇偶：
  - 偶数：生成宽元素（W），`value = (value - 2) / 2`
  - 奇数：生成窄元素（N），`value = (value - 1) / 2`
- 收集的序列反转，使最低有效位在右侧。
- 渲染规则：
  - N = 窄 bar（1 模块）+ space（2 模块）→ 模式 `12`
  - W = 宽 bar（3 模块）+ space（2 模块）→ 模式 `32`
- 符号以 bar 开始和结束。

### 解码规则
- 提取中间行，归一化宽度。
- 奇数个 group，首尾为 bar。
- 每对 bar/space 宽度为 1 或 2。
- 映射：
  | bar | space | bit |
  |:---:|:-----:|:---:|
  | 1 | 1 | 0 |
  | 1 | 2 | 1 |
  | 2 | 1 | 2 |
  | 2 | 2 | 3 |
- 从右向左解析为四进制数。

## 2. Plessey（Original / UK Plessey）

### 编码规则
- 字符集：十六进制 `0-9 A-F`。
- 起始符：`21211221`
- 每个十六进制数字编码为 4 位；bit 0 → `12`，bit 1 → `21`。
- 计算 CRC-8 校验位，生成多项式：`1 + x + x^2 + x^3 + x^5 + x^8`。
- 附加 8 位 CRC（低 nibble 在前，高 nibble 在后）。
- 终止符：`201` + 反转起始符 `21211212`。

### 解码规则
- 至少 20 个 group。
- 每 4 位解析为一个十六进制字符，第 5 位为分隔符。
- 返回长度 ≥4 的十六进制字符串。
- 当前实现不严格校验 CRC。

## 3. MSI Plessey

### 编码规则
- 字符集：数字 `0-9`。
- 起始符：`21`
- 每个数字用 8 模块模式表示（例如 `0 → 12121212`，`1 → 12121221`）。
- 可选校验位：
  - 一位 mod-10
  - 两位 mod-10
  - 一位 mod-11
  - mod-11 后 mod-10
- 停止符：`121`

### 解码规则（新增）
- 识别 `21` 起始符和 `121` 停止符。
- 中间每 8 模块解析为一个数字。
- 校验 mod-10 或 mod-11。

---

## 4. Telepen

### 编码规则
- 字符集：完整 ASCII（0x00-0x7F）。
- 每个字符为固定 16 模块模式。
- 起始符：`'_'`（ASCII 95）的模式。
- 对每个输入字符追加其 16 模块模式。
- 校验位：`check = 127 - (sum % 127)`，若为 127 则取 0。
- 停止符：`'z'`（ASCII 122）的模式。
- 另有 Telepen Numeric 模式，将数字对压缩为更短模式。

### 解码规则
- 偶数个 group，至少 16 个。
- 每 8 个交替 bar/space 组成一个字符。
- 每个字符总宽度为 8 模块。
- bar=1，space=0，转换为字符。

---

## 5. 渲染约定

- 模块宽度至少 2px，避免舍入失真。
- 一维码高度通常为 80-150px。
- 使用 `BarcodeScanUtils.extractBars(bitmap)` 读取中间行进行解码验证。
